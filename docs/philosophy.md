# PRISM — 理念体系

## 核心命题

> PRISM 是一个氛围设备。它不试图成为最强大的工具，而是成为最合适的存在。

---

## 存在感哲学

PRISM 不是手机的替代品，也不是手表的竞争者。
它是一个"第三空间"设备——当你不想被手机绑架，但又需要和数字世界保持连接的时候，PRISM 在那里。

它的存在感是克制的：
- 不主动打扰
- 不强迫交互
- 但随时准备好

拿起它，它就醒来。放下它，它安静地呼吸。

---

## 信息哲学

**不是所有信息都值得显示。**

PRISM 的屏幕在任何时刻只呈现一件最重要的事。
其余的信息在需要时才出现，不需要时消失。

这不是功能限制，这是对注意力的尊重。

信息层级：
1. 此刻最重要的事（全屏主角）
2. 环境状态（角标、灯光，不占主视野）
3. 历史和次要信息（需要主动查询才出现）

---

## 反馈哲学

每一个操作都有回应，但回应是克制的。

不是弹窗，不是震动，而是：
- 一个灯光的变化
- 一个数字从下方冒出
- 一个菜单以弹性滚入

反馈存在，但不打断心流。
用户感知到了，但不需要停下来处理它。

**灯光先于屏幕响应。** 灯光的延迟更低，更接近直觉。

---

## 姿态哲学

PRISM 感知你怎么拿它，并相应地改变自己。

横持是工作模式，竖持是信息模式，翻转是静默模式。
这些不是设置项，是自然的肢体语言。

设备应该适应人，而不是人适应设备。

---

## 工艺哲学

软件和硬件都是产品的一部分。

- 代码的结构要像电路板的布线一样整洁
- UI 的像素要像外壳的倒角一样精确
- 库的接口要像按键的手感一样顺滑

每一个可以被单独抽出来复用的模块，都应该被抽出来做成库。
好的工艺不藏在产品里，它会自然地流出来。

---

## 开发原则

这些理念直接指导开发决策：

| 理念 | 对应决策 |
|------|---------|
| 信息克制 | 主屏只显示一件事，不堆砌 widget |
| 反馈克制 | 动画 120~200ms，不拖沓，不炫技 |
| 姿态适应 | MPU6050 驱动 UI 旋转，不是设置项 |
| 灯光优先 | 状态变化先更新灯光，再更新屏幕 |
| 工艺外溢 | 可复用模块独立成库（NodeLib 体系） |
| 心流保护 | 任何交互不强制打断当前状态 |
